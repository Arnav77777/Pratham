About the Event

Our event is designed to bring together industry professionals, students, and enthusiasts to discuss the latest trends in the field.

Competition Format

Rules

1. All code development must be done prior to the event day. No coding or modifications are allowed during the event.
2. Participants can use any programming language or technology, such as C, C++, Java, Python, HTML, CSS, or JavaScript.
3. Participants will be grouped into clusters based on the programming language used, with winners selected from each cluster.
4. The code should be brought to the event on a USB drive or any other portable storage device, or own laptops and the code must be ready to run on the day of the event
5. Each group / participant will be given 10-15 minutes to present their work to the judges.
6. The presentation should include a brief introduction to the technology, explanation of the code, and a live demonstration.
7. All code must be original and created solely by the participant. Plagiarism or copying code from external sources is strictly prohibited.
8. Any form of cheating, including collaboration with others, using unauthorized resources, or altering the code after submission, is not allowed and will result in immediate disqualification.
9. Participants must arrive at the venue at least 30 minutes before their scheduled presentation time.
10. Disruptive behaviour, cheating, or any form of malpractice will result in immediate disqualification.
